概要

アイテムのデータを保持するActorComponent
InventoryComponentを持つアクターには、PtoInventorySystemInterfaceを追加して、 外部からInventoryComponentを取得できるようにする必要があります。

特徴

  • Actorに複数のInventoryComponentを持たせて、 それぞれで違うUIを持たせたり、処理を行うことができます。

  • InventoryComponentを特定のPrimaryAssetType専用にすることができます。

    例えばPotionItemというPrimaryAssetTypeを作った場合、

    プレイヤーがPotionItemを持ったアイテムを獲得した際に、GetInventoryCompByAssetIdを用いて InventoryComponentを取得し、それにPotionItemを追加する。

    この処理をフィルターとして、PotionItem専用のInventoryComponentとみなすことができます。
